Angular Boilerplate

Getting Started

This boilerplate is a GitHub template. If you plan to store your project on GitHub you can follow the procedures documented on GitHub: Creating a repository from a template.

If you do not plan to store your project on GitHub: 

			git clone https://github.com/keithstric/angular-boilerplate-plain new-project
			cd new-project
			rm .git
			npm install
			npm start

Configure for YOUR project

You will want to configure the boilerplate to meet your needs. You may want to change all or just some of the following items. Hopefully future versions can take care of some of this automatically.

  • package.json - The name field. This is used to drive the site title, header title and localStorage/sessionStorage prefix. If you create your project from the template on GitHub, this step may not be required.
  • src/app/core/interfaces/api.interface.ts - Update the ApiEndpoints enum and ApiRouteToClass constant to match your routes
  • proxy.conf.json - Modify to match your backend api location and port. You can delete this file if there is no need to proxy routes. Be sure to update angular.json if you delete this file
  • Breadcrumbs - If not using breadcrumbs delete the following files:
    • src/app/layout/components/breadcrumbs
    • src/app/layout/components/page-breadcrumb-header
    • src/app/layout/services/breadcrumb
  • src/scss/theme.scss - To meet your theming needs
  • Authenticaton - Update the following for your authentication needs
    • src/app/modules/auth
    • src/app/core/services/auth/auth.service.ts
  • src/app/app-routing.module.ts - Remove the following modules and update
    • src/app/modules/home
    • src/app/modules/about
    • src/app/modules/features
  • src/app/app.module.ts - Remove above modules
